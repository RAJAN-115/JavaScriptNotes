what is javascript

JavaScript is a programming language used to make websites interactive and dynamic. It allows web pages to do more than just display static information. With JavaScript, you can add features like animations, form validations, and interactive elements that respond to user actions, such as clicks or mouse movements. It's easy to learn and widely used across the web.

Dynamic in web development refers to web pages that can change and update in real-time based on user interactions or other events, without needing to reload the entire page.

JavaScript is an interpreted language, meaning it runs directly in the browser without needing to be compiled first. This makes it easy to test and debug. Additionally, JavaScript has object-oriented features, allowing developers to structure code using objects. This helps in creating organized, reusable, and maintainable code.

JavaScript History:

  • 1995: JavaScript was developed by Brendan Eich at Netscape.
  • Initial Names: Originally named Mocha, it was briefly renamed to LiveScript before finally becoming JavaScript.
  • Standardization: The language was standardized as ECMAScript by ECMA International to ensure compatibility across different browsers and platforms.

When a user enters a website URL, the browser sends a request to the server. The server responds with HTML (structure), CSS (design), and JavaScript (interactivity). The browser then renders the page, and JavaScript enables dynamic interactions without reloading. This process makes websites functional and responsive

Advantages of JavaScript

  1. Client-Side Execution
    • JavaScript runs directly in the user's browser, which means it doesn’t need to communicate with the server for every task.
    • Benefit : Faster response time and reduced load on the server.
    • Example : Clicking a button to show or hide content happens instantly without reloading the page.
  1. Validation on Browser
    • JavaScript can validate user input (like forms) before sending it to the server.
    • Benefit : Saves time and bandwidth by catching errors early.
    • Example : If a user forgets to fill in a required field, JavaScript can immediately alert them instead of waiting for the server to respond.
  1. Easy Language
    • JavaScript is beginner-friendly and has a simple syntax.
    • Benefit : Easy to learn for new developers and widely supported across all browsers.
    • Example : You don’t need complex setup or tools to start coding—just open a browser and start writing JavaScript!

Disadvantages of JavaScript:

  1. Less Secure:
    • Description: JavaScript code runs on the client side, making it visible to users. This can expose sensitive logic and make it vulnerable to attacks like cross-site scripting (XSS).
    • Example: Malicious users can view and potentially manipulate JavaScript code, posing security risks.
  1. No Hardware Access:
    • Description: JavaScript running in a browser has limited access to the user's hardware for security reasons. This means it can't directly interact with hardware components like the file system or camera without specific permissions.
    • Example: To access a user's webcam or files, JavaScript needs explicit user permission through browser APIs.
  1. JavaScript-Enabled Browsers:
    • Description: JavaScript relies on the browser to execute, so it requires users to have JavaScript enabled. If JavaScript is disabled, interactive features won't work.
    • Example: Some users may disable JavaScript for security or privacy reasons, which can break functionality on websites that depend on it.

Ways of Adding JavaScript

There are two main ways to include JavaScript in your HTML file:

  1. Inline JavaScript

    This means writing JavaScript code directly inside your HTML file using the <script> tag.

    • Inside <head> Tag:

      You can place the <script> tag inside the <head> section of your HTML file.

      Example:

      <html>
  <head>
    <title>Hello JS</title>
    <script type="text/javascript">
      document.write("Hello GeekyShows");
    </script>
  </head>
  <body>
    <h1>I am Heading</h1>
    <p>I am first Paragraph.</p>
  </body>
</html>
    • Inside <body> Tag:

      You can also place the <script> tag inside the <body> section, usually at the end, so that the script runs after the HTML content is loaded.

      Example:

      <html>
  <head>
    <title>Hello JS</title>
  </head>
  <body>
    <h1>I am Heading</h1>
    <p>I am first Paragraph.</p>
    <script type="text/javascript">
      document.write("Hello GeekyShows");
    </script>
  </body>
</html>
  1. External JavaScript File

    Instead of writing JavaScript code directly in the HTML file, you can store it in an external .js file and link it to your HTML file.

    • Inside <head> Tag:

      Link the external JavaScript file inside the <head> section using the src attribute in the <script> tag.

      Example:

      <html>
  <head>
    <title>Hello JS</title>
    <script src="geek.js" type="text/javascript"></script>
  </head>
  <body>
    <h1>I am Heading</h1>
    <p>I am first Paragraph.</p>
  </body>
</html>
    • Inside <body> Tag:

      You can also link the external JavaScript file inside the <body> section, usually at the end.

      Example:

      <html>
  <head>
    <title>Hello JS</title>
  </head>
  <body>
    <h1>I am Heading</h1>
    <p>I am first Paragraph.</p>
    <script src="geek.js" type="text/javascript"></script>
  </body>
</html>

Key Points to Remember

  • Inline JavaScript: Good for small scripts or quick testing but not recommended for larger projects.
  • External JavaScript: Best practice for organizing and reusing code. Keeps your HTML clean and separates logic from structure.
  • Always use the <script> tag with the src attribute to link external JavaScript files (e.g., src="geek.js").
  • Place scripts at the end of the <body> tag when possible to ensure the HTML content loads first.

Why Should You Remember This?

You can confidently explain it like this:

  • "JavaScript can be added inline directly in the HTML file using the <script> tag, either in the <head> or <body>. For better organization, we can also use external JavaScript files by linking them with the src attribute in the <script> tag."

Key Components of <script> Tag

  1. <script> - Opening Script Tag
    • This is where JavaScript code begins in an HTML file. It tells the browser that the content inside is JavaScript.
  1. src Attribute
    • The src attribute specifies the location (source) of an external JavaScript file.
    • Example:
      Here,
      geek.js is the external JavaScript file being linked to the HTML file.
      <script src="geek.js"></script>
  1. File Name and Extension
    • geek.js:
      • geek is the name of the JavaScript file.
      • .js is the file extension, which indicates it’s a JavaScript file.
  1. type Attribute
    • The type attribute tells the browser what type of script is being used.
    • Example:
      <script type="text/javascript"></script>
      • Nowadays, the type attribute is optional because browsers assume text/javascript by default.
  1. text/javascript
    • This specifies that the script is written in JavaScript.
    • Modern browsers automatically assume this, so you don’t need to include it explicitly.
  1. document.write() Function
    • This function is used to display data on the webpage.
    • Example:
      document.write("Hello World");
      • This will print "Hello World" on the webpage.
      • Note: Avoid using document.write() in real projects because it can overwrite the entire page if used after the page has loaded.
  1. </script> - Closing Script Tag
    • This marks the end of the JavaScript code block.
    • Example:
      <script>
  document.write("Hello World");
</script>

Summary

  • <script> starts the JavaScript section.
  • src links an external JavaScript file (e.g., geek.js).
  • type="text/javascript" is optional in modern browsers.
  • document.write() displays data on the webpage.
  • </script> ends the JavaScript section.

Key Takeaways from the Concept

✅ Including Multiple JavaScript Files

  • Use multiple <script> tags with src to include external JS files.
  • Example:
    <script src="geek1.js"></script>
<script src="geek2.js"></script>
<script src="geek3.js"></script>

✅ Order of Execution Matters

  • JavaScript files execute sequentially in the order they appear.
  • If one script depends on another, load them in the correct sequence.

✅ Place Scripts at the End of <body>

  • Ensures the HTML content loads first, preventing errors.
  • Improves performance and user experience.

✅ Avoid document.write()

  • It can overwrite the entire document if used after page load.
  • Instead, use console.log() or innerHTML.

✅ Best Practices for Modern JavaScript

  • Use ES6 Modules (import/export) for modular code.
  • Consider using async or defer attributes:
    <script src="script.js" defer></script>
    • async → Loads the script without blocking the page rendering.
    • defer → Loads the script after the document is fully parsed.

Final Thought

Using external JS files keeps your HTML clean, makes debugging easier, and promotes reusability! 🚀

document.write() in JavaScript

The document.write() method is a built-in JavaScript function used to write or display content directly on an HTML page. It dynamically adds text, HTML elements, or other data to the webpage.

Key Points About document.write()

  1. Purpose:
    • Used to write arbitrary HTML and content into the webpage.
    • Often used for testing purposes or quick debugging.
  1. Syntax:
    document.write(content);
    • content: The text, HTML, or dynamic data you want to display on the page.
  1. Examples:
    • Writing plain text:
      document.write("Hello World");

      Output: Displays "Hello World" on the webpage.

    • Writing HTML:
      document.write("<h1>Welcome to GeekyShows</h1>");

      Output: Displays a heading "Welcome to GeekyShows".

    • Writing dynamic values:
      var name = "GeekyShows";
document.write("Hello " + name);

      Output: Displays "Hello GeekyShows".

    • Writing multiple lines:
      document.write("Line 1 <br>");
document.write("Line 2 <br>");

      Output:

      Line 1
Line 2
  1. Behavior:
    • If used after the page has fully loaded, document.write() will overwrite the entire document (clearing all existing content).
    • Example:
      Output: The
      <h1> element will be replaced with "Overwritten!".
      <html>
  <body>
    <h1>Hello</h1>
    <script>
      document.write("Overwritten!");
    </script>
  </body>
</html>
  1. Best Practices:
    • Avoid using document.write() in real-world projects because it can cause unexpected behavior, especially after the page has loaded.
    • Use modern DOM manipulation methods like innerHTML, appendChild, or insertAdjacentHTML instead.

Advantages

  • Simple and easy to use for beginners.
  • Useful for quick testing or debugging.

Disadvantages

  • Overwrites the entire document if used after the page loads.
  • Not recommended for production code due to potential performance and usability issues.

Alternatives to document.write()

Instead of document.write(), consider using these modern approaches:

  1. Using innerHTML:
    document.getElementById("demo").innerHTML = "Hello World";

    This updates the content of a specific element without affecting the rest of the page.

  1. Using textContent:
    document.getElementById("demo").textContent = "Hello World";

    This updates only the text content of an element.

  1. Using console.log():
    For debugging purposes, use:
    console.log("This is a log message");

When to Use document.write()

  • Only for simple testing or learning purposes.
  • Avoid using it in production websites or applications.

window.alert() in JavaScript:

The window.alert() function is a built-in JavaScript method used to display information or messages to the user in a pop-up dialog box. It is commonly used for debugging, warnings, or notifying users about something important.

Key Points About window.alert()

  1. Purpose:
    • Displays a message in a small dialog box.
    • The dialog box contains an "OK" button that the user must click to close it.
    • Execution of the script pauses until the user interacts with the alert box (i.e., clicks "OK").
  1. Syntax:
    window.alert(message);
    • message: The content to be displayed in the alert box. It can be a string, number, variable, or expression.
  1. Examples:
    window.alert("Hello World"); // Displays "Hello World"
window.alert(variable);      // Displays the value of the variable
window.alert(4 + 2);         // Displays the result of the expression: "6"
window.alert("Hello World " + variable); // Concatenates text and variable value
  1. Behavior:
    • The alert box is modal, meaning it blocks further interaction with the webpage until the user acknowledges it by clicking "OK."
    • It does not return any value; its purpose is purely to display information.
  1. Best Practices:
    • Use alert() sparingly, as frequent interruptions can frustrate users.
    • Avoid using alert() for debugging in production code. Instead, use console.log() for logging messages to the browser's console.
  1. Example in Context:
    <script>
    var name = "GeekyShows";
    window.alert("Welcome to " + name); // Displays "Welcome to GeekyShows"
    window.alert(10 + 20);             // Displays "30"
</script>

Why Use window.alert()?

  • User Attention: When you need to grab the user's attention immediately.
  • Simple Debugging: For quick testing during development (though console.log() is preferred for debugging).
  • Notifications: To inform users about errors, warnings, or important updates.

Limitations:

  • Blocking Nature: Pauses the execution of the script and prevents user interaction with the page.
  • No Styling: The appearance of the alert box cannot be customized.
  • Not User-Friendly: Overusing alerts can disrupt the user experience.

Alternatives to window.alert():

  • console.log(): For debugging purposes, outputs messages to the browser's console without interrupting the user.
  • Custom Modal Dialogs: Use HTML, CSS, and JavaScript to create styled, non-blocking dialog boxes.
  • confirm(): Displays a dialog box with "OK" and "Cancel" buttons for user confirmation.
  • prompt(): Displays a dialog box that asks the user for input.

Example Output:

If the following code is executed:

var name = "John";
window.alert("Hello " + name); // Displays "Hello John"
window.alert(10 + 20);        // Displays "30"

The user will see two alert boxes:

  1. First box: "Hello John"
  1. Second box: "30"

Identifiers in JavaScript:

An identifier is a name used to identify variables, functions, constants, or other entities in JavaScript. Identifiers are fundamental to writing code because they allow you to refer to these entities throughout your program.

Key Points About Identifiers

  1. Purpose:
    • Identifiers are used to name variables, functions, constants, and other elements in your code.
    • They help make your code readable and maintainable by giving meaningful names to entities.
  1. Rules for Naming Identifiers:
    • Allowed Characters:
      • Letters (A-Z or a-z)
      • Digits (0-9)
      • Underscore (_)
      • Dollar sign ($)
    • Restrictions:
      • An identifier must begin with a letter (A-Z or a-z), an underscore (_), or a dollar sign ($).
      • It cannot start with a digit (e.g., 2variable is invalid).
      • It cannot contain spaces.
      • It cannot be a reserved keyword (e.g., var, if, else, etc.).
    • Case Sensitivity:
      • JavaScript is case-sensitive, so myVariable and myvariable are treated as different identifiers.
  1. Examples of Valid Identifiers:
    X2       // Valid: Starts with a letter
PI       // Valid: Uppercase letters are allowed
Sigma    // Valid: Mixed-case letters are allowed
matadd   // Valid: Lowercase letters are allowed
_temp    // Valid: Starts with an underscore
$price   // Valid: Starts with a dollar sign
  1. Examples of Invalid Identifiers:
    2variable  // Invalid: Cannot start with a digit
my var     // Invalid: Cannot contain spaces
var        // Invalid: Reserved keyword
  1. Best Practices:
    • Use meaningful names for identifiers to improve code readability.
    • Follow consistent naming conventions:
      • Use camelCase for variables and functions (e.g., firstName, calculateSum).
      • Use UPPERCASE for constants (e.g., PI, MAX_VALUE).
  1. Examples in Context:
    var X2 = 10;           // A variable named X2
const PI = 3.14;       // A constant named PI
function Sigma() {     // A function named Sigma
    console.log("Hello");
}
let matadd = 5 + 3;    // A variable named matadd

Why Are Identifiers Important?

  • Readability: Good identifiers make your code easier to understand.
  • Reusability: Identifiers allow you to reuse variables, functions, and constants throughout your program.
  • Maintainability: Clear and consistent naming conventions make it easier to debug and update your code.

Common Mistakes to Avoid

  1. Using reserved keywords as identifiers:
    var if = 10; // Error: "if" is a reserved keyword
  1. Starting identifiers with numbers:
    var 2ndPlace = "Silver"; // Error: Cannot start with a digit
  1. Using special characters other than _ or $:
    var @email = "test@example.com"; // Error: "@" is not allowed

Variables in JavaScript:

A variable is a named storage location in memory that holds a value. Variables are used to store data that can be accessed and manipulated throughout your program. In JavaScript, variables are declared using identifiers (names) that follow specific rules.

Key Points About Variables

  1. Definition:
    • A variable is an identifier or name that refers to a value.
    • It acts as a container for storing data such as numbers, strings, objects, etc.
  1. Naming Rules:
    • Allowed Characters:
      • Letters (A-Z or a-z)
      • Digits (0-9)
      • Underscore (_)
      • Dollar sign ($)
    • Restrictions:
      • The first character must be a letter (A-Z or a-z), an underscore (_), or a dollar sign ($).
      • A variable name cannot start with a number.
      • Variable names cannot contain spaces.
      • Variable names cannot be reserved keywords (e.g., var, if, else, etc.).
    • Case Sensitivity:
      • JavaScript is case-sensitive, so myVariable and myvariable are treated as different variables.
  1. Examples of Valid Variable Names:
    c
fact
b33
total_amount
_temp
$price
  1. Examples of Invalid Variable Names:
    58show       // Invalid: Cannot start with a number
Steel City   // Invalid: Cannot contain spaces
var          // Invalid: Reserved keyword
  1. Declaration:
    • Variables are declared using the var, let, or const keywords:
      • var: Used in older versions of JavaScript (function-scoped).
      • let: Introduced in ES6, allows block-scoped variables.
      • const: Used for constants whose values cannot be reassigned.
    var x = 10;         // Function-scoped
let y = 20;         // Block-scoped
const z = 30;       // Constant, cannot be reassigned
  1. Initialization:
    • A variable can be declared without being initialized, in which case its value is undefined.
    • You can also assign a value to a variable during declaration.
    let roll;           // Declared but not initialized (value is undefined)
roll = 101;         // Initialized later

let name = "John";  // Declared and initialized in one step
  1. Dynamic Typing:
    • JavaScript is dynamically typed, meaning you don't need to specify the type of a variable explicitly.
    • A variable can hold any type of data (e.g., string, number, boolean, object, etc.).
    let num = 42;               // Number
let text = "Hello World";   // String
let isActive = true;        // Boolean

Example Usage

// Declaring variables
let c = 10;
let fact = "JavaScript is fun!";
let b33 = 3.14;
let total_amount = 1000;

// Displaying variable values
console.log(c);           // Output: 10
console.log(fact);        // Output: JavaScript is fun!
console.log(b33);         // Output: 3.14
console.log(total_amount); // Output: 1000

Best Practices

  • Use meaningful names for variables to improve code readability.
  • Follow consistent naming conventions:
    • Use camelCase for variable names (e.g., totalAmount, userName).
    • Use uppercase for constants (e.g., PI, MAX_VALUE).
  • Avoid using single-letter variable names unless they are temporary or loop counters.

Why Are Variables Important?

  • Data Storage: Variables allow you to store and reuse data throughout your program.
  • Flexibility: They make your code dynamic by allowing values to change during execution.
  • Readability: Properly named variables make your code easier to understand and maintain.

Function

A function in JavaScript is a block of reusable code that is designed to perform a specific task. Functions are essential for organizing code into manageable and reusable pieces. They can compute values, perform actions, or both.

Types of Functions:

  1. Library or Built-in Functions :
    These are pre-defined functions provided by JavaScript. You don't need to create them; you just call them when needed.

    Examples:

    • valueOf(): Returns the primitive value of a variable.
    • document.write(): Writes content directly to the HTML document.
    • alert(): Displays an alert box with a message.
  1. User-defined Functions :
    These are functions created by the programmer to perform custom tasks. They allow you to define your own logic and reuse it wherever necessary.

Creating and Calling a Function

Creating a Function :

You define a function using the function keyword, followed by the function name, parentheses (), and curly braces {} containing the function's code block.

Syntax:

function functionName(parameters) {

// Block of statements

}

Example:

function greet(name) {

document.write("Hello, " + name);

}

Calling a Function :

Once a function is defined, you can execute it by calling its name followed by parentheses ().

Syntax:

functionName(arguments);

Example:

greet("Alice"); // Output: Hello, Alice

Function Name Rules :

  • The function name must start with a letter, underscore (_), or dollar sign ($).
  • It cannot start with a number.
  • Avoid using reserved keywords (e.g., else, if, etc.).
  • Function names are case-sensitive.

Execution Flow Example

Consider the following code:

document.write("First Line <br />");

display();

document.write("GeekyShows");

function display() {

document.write("Welcome to GeekyShows <br />");

}

document.write("Last Line <br />");

Output:

First Line

Welcome to GeekyShows

GeekyShows

Last Line

Explanation:

  • The first document.write("First Line <br />"); is executed, printing "First Line".
  • Then, the display() function is called. The interpreter jumps to the display function and executes its code, printing "Welcome to GeekyShows".
  • After the function finishes executing, control returns to the next line after the function call, which is document.write("GeekyShows");, so "GeekyShows" is printed.
  • Finally, the last document.write("Last Line <br />"); is executed, printing "Last Line".

4. How Function Calls Work Internally

  • Call Stack: When a function is called, it is added to the call stack . The call stack keeps track of where the program is in its execution. Once the function finishes executing, it is removed from the call stack, and the program continues with the next line of code.
  • Scope: When a function is called, a new scope is created for that function. Variables declared inside the function are local to that function and cannot be accessed outside of it.
  • Return Value: If the function has a return statement, the value specified in the return statement is returned to the caller. If there is no return statement, the function returns undefined.

Functions with Parameters

In JavaScript, functions can accept parameters (also called arguments) to make them more flexible and reusable. Parameters allow you to pass data into a function, which the function can then use to perform operations or computations.

Syntax:

function function_name(parameter1, parameter2, ...) {
    // Block of statements
}

Example:

function display(name) {
    document.write(name);
}

In this example:

  • The function display takes one parameter, name.
  • When the function is called, it writes the value of name to the document.

Key Points About Function Parameters in JavaScript:

  1. No Data Type Specification:
    • JavaScript does not require you to specify the data type of parameters when defining a function.
    • For example, you don't need to declare whether a parameter is a string, number, or object.
    function add(a, b) {
    return a + b;
}

    In this case, a and b could be numbers, strings, or other types. JavaScript will handle them dynamically.

  1. No Type Checking:
    • JavaScript does not enforce type checking on the arguments passed to a function.

global and local scope

Global Scope

A variable declared outside any function is a global variable. It is accessible and modifiable throughout your entire program. In a web browser, global variables are deleted when you close the browser window (or tab), but they remain available to new pages loaded into the same window.

Example:

  • The add function can access and use the global variable a.
  • When add(20) is called, it returns 30 because a is 10 and b is 20.

Local Scope

A variable declared inside a function is a local variable. It is created and destroyed every time the function is executed, and it cannot be accessed by any code outside the function. If a variable inside a function is not declared with var, let, or const, it becomes a global variable.

Example 1: Local Variable

}

document.write(add(20)); // Output: 30
  • a is declared inside the add function, so it is a local variable.
  • It is created when the function is called and destroyed when the function exits.
  • It cannot be accessed outside the function.

    a = 10; // Global variable (because `var` is not used)
    return a + b;
}

document.write(add(20)); // Output: 30
console.log(a); // Output: 10 (`a` is now a global variable)
  • a is not declared with var, so it becomes a global variable.
  • It can be accessed outside the function.

Variable Hoisting

Hoisting is a JavaScript mechanism where variable and function declarations are moved ("hoisted") to the top of their containing scope during the compile phase, before the code is executed. This means that no matter where variables or functions are declared in the code, they are treated as if they were declared at the top of their scope (either global or function scope).

However, it's important to note that only the declarations are hoisted, not the initializations. This can lead to some unexpected behavior if you're not careful.

Key Concepts of Variable Hoisting

  1. var Hoisting:
    • Variables declared with var are hoisted to the top of their scope, but they are initialized with undefined.
    • This means you can access the variable before its declaration, but its value will be undefined.

    Example:

    console.log(x); // Output: undefined
var x = 10;

    Explanation:
    The above code is interpreted by JavaScript as:

    var x;          // Declaration is hoisted
console.log(x); // undefined (because initialization happens later)
x = 10;         // Initialization remains in place
  1. let and const Hoisting:
    • Variables declared with let and const are also hoisted, but they are not initialized with undefined. Instead, they enter what's called the "Temporal Dead Zone" (TDZ).
    • You cannot access the variable before its declaration; doing so will result in a ReferenceError.

    Example with let:

    console.log(y); // ReferenceError: Cannot access 'y' before initialization
let y = 20;

    Example with const:

    console.log(z); // ReferenceError: Cannot access 'z' before initialization
const z = 30;

    Explanation:

    • Both let and const are hoisted, but they are not accessible until the line where they are declared is reached. This is why accessing them before declaration results in an error.
  1. Function Hoisting:
    • Function declarations are also hoisted, and they are fully initialized before the code runs. This means you can call a function before its declaration in the code.
    • However, function expressions (when assigned to variables) are not hoisted in the same way.

    Example of Function Declaration Hoisting:

    sayHello(); // Output: "Hello!"
function sayHello() {
    console.log("Hello!");
}

    Example of Function Expression (Not Hoisted):

    sayHi(); // TypeError: sayHi is not a function
var sayHi = function() {
    console.log("Hi!");
};

    Explanation:

    • In the second example, sayHi is treated like a variable declared with var, so it is hoisted but initialized as undefined. When you try to call it as a function, it throws a TypeError.

Why Does Hoisting Happen?

JavaScript hoisting occurs because of how the JavaScript engine processes your code. During the creation phase of the execution context (before the code is executed), the engine scans the code for variable and function declarations and allocates memory for them. This is why declarations are "hoisted" to the top of their scope.

Best Practices to Avoid Hoisting Issues

  1. Always Declare Variables Before Using Them:
    • To avoid confusion and potential bugs, always declare your variables (var, let, or const) before using them.
    let x = 10;
console.log(x); // Output: 10
  1. Use let and const Instead of var:
    • Since let and const are block-scoped and throw errors when accessed before declaration, they help prevent accidental hoisting issues.
    const y = 20;
console.log(y); // Output: 20
  1. Avoid Relying on Hoisting:
    • Even though hoisting is a feature of JavaScript, relying on it can make your code harder to read and maintain. Always write your code in a way that makes the flow of execution clear.

Conclusion

Hoisting is a fundamental concept in JavaScript that affects how variables and functions are processed. While it can sometimes lead to unexpected behavior, understanding how hoisting works allows you to write more predictable and bug-free code. By following best practices like declaring variables before use and preferring let and const over var, you can minimize the risks associated with hoisting.

Closure

A closure is a fundamental concept in JavaScript that allows an inner function to access variables from its outer (enclosing) function, even after the outer function has finished executing. In simpler terms, a closure "remembers" the environment in which it was created, preserving access to variables from its parent scope.

Key Characteristics of Closures:

  1. Access to Outer Function's Variables:
    • A closure can access variables from its own scope, the outer function's scope, and the global scope.
  1. Preservation of Data:
    • Even after the outer function has completed execution, the inner function (closure) retains access to the outer function's variables.
  1. Three Scopes in a Closure:
    • Local Scope: The variables defined within the inner function itself.
    • Outer Function's Scope: The variables defined in the enclosing (outer) function.
    • Global Scope: Variables defined globally, outside of any function.

Example of a Closure

function outerFunction(outerVariable) {
    return function innerFunction(innerVariable) {
        console.log('Outer Variable:', outerVariable);
        console.log('Inner Variable:', innerVariable);
    }
}

const closureFunction = outerFunction('outside');
closureFunction('inside');

Explanation:

  • outerFunction takes a parameter outerVariable and returns an innerFunction.
  • When outerFunction('outside') is called, it returns the innerFunction, which is stored in the variable closureFunction.
  • Even though outerFunction has finished executing, the innerFunction (now stored in closureFunction) still has access to the outerVariable ('outside') because of the closure.
  • When closureFunction('inside') is called, it logs both the outerVariable ('outside') and the innerVariable ('inside').

Output:

Outer Variable: outside
Inner Variable: inside

Why Use Closures?

  1. Data Encapsulation: Closures allow you to create private variables that cannot be accessed directly from outside the function, but can be manipulated through the closure.
  1. Function Factories: You can use closures to create functions that generate other functions with specific behaviors.
  1. Maintaining State: Closures are useful for maintaining state in asynchronous operations or when working with callbacks.

Example: Using Closure for Data Encapsulation

function createCounter() {
    let count = 0;  // This variable is private and cannot be accessed directly
    return {
        increment: function() {
            count++;
            console.log(count);
        },
        decrement: function() {
            count--;
            console.log(count);
        }
    };
}

const counter = createCounter();
counter.increment();  // Output: 1
counter.increment();  // Output: 2
counter.decrement();  // Output: 1

Explanation:

  • The createCounter function returns an object with two methods: increment and decrement.
  • The count variable is encapsulated within the createCounter function and cannot be accessed directly from outside.
  • However, the increment and decrement methods have access to the count variable due to the closure, allowing them to modify it.

Example: Function Factory Using Closure

function multiplier(factor) {
    return function(number) {
        return number * factor;
    }
}

const double = multiplier(2);
const triple = multiplier(3);

console.log(double(5));  // Output: 10
console.log(triple(5));  // Output: 15

Explanation:

  • The multiplier function takes a factor and returns a new function that multiplies any number by that factor.
  • double and triple are closures that "remember" the factor value passed to multiplier.
  • When double(5) is called, it multiplies 5 by 2 (the remembered factor), resulting in 10.

Key Points to Remember:

  1. Closures Preserve Scope: A closure retains access to its outer function's variables even after the outer function has returned.
  1. Private Variables: Closures can be used to create private variables that are inaccessible from outside the function.
  1. Avoid Memory Leaks: Since closures keep references to their outer scope, they can sometimes lead to memory leaks if not managed properly. Always be cautious when using closures in long-running applications.

Conclusion

Closures are a powerful feature in JavaScript that allow functions to "remember" the environment in which they were created. They are widely used for data encapsulation, creating function factories, and maintaining state in asynchronous operations. Understanding closures is crucial for mastering advanced JavaScript concepts like module patterns, currying, and higher-order functions.

Function Expression

A function expression is a way to define a function in JavaScript by assigning it to a variable. Unlike function declarations, function expressions are not hoisted, meaning they cannot be invoked before they are defined.

Key Characteristics of Function Expressions

  1. Definition and Assignment:

    • The function can be anonymous (without a name) or named.
  1. Invocation:
    • The function can only be called after it has been defined and assigned to the variable.
    • Attempting to call the function before its definition will result in a ReferenceError.
  1. Hoisting Behavior:

  • The function can now be invoked using myfun() after the assignment.

Immediately Invoked Function Expressions (IIFE)

An IIFE (Immediately Invoked Function Expression) is a JavaScript function that runs as soon as it is defined. This pattern is useful for avoiding global variable pollution, creating private scopes, and executing code immediately without needing to call the function later.

Key Features of IIFE

  1. Immediate Execution: The function is executed right after it is created.
  1. Avoids Global Scope Pollution: Variables declared inside the IIFE are not accessible outside, preventing conflicts with other parts of your code.
  1. Private Scope: You can create private variables and functions that cannot be accessed from outside the IIFE.
  1. Self-Contained: IIFEs are self-contained units of code, making them ideal for initialization tasks or modular code.

Basic Syntax

(function() {
    // Code to be executed immediately
})();
  • Explanation:
    • The function is wrapped in parentheses (function() { ... }) to make it an expression.
    • The () at the end immediately invokes the function.

Examples

1. Basic IIFE

This example demonstrates a simple IIFE that writes "GeekyShows" to the document:

(function() {
    document.write("GeekyShows");
})();
// Output: GeekyShows
  • How It Works:
    • The function is defined and immediately invoked.
    • The message "GeekyShows" is written to the document.

2. IIFE with Parameters

You can pass parameters to an IIFE, just like any other function:

(function(a, b) {
    document.write(a + " " + b);
})(10, 20);
// Output: 10 20
  • How It Works:
    • The function takes two parameters, a and b.
    • It is immediately called with the values 10 and 20.
    • The output is 10 20.

3. Avoiding Global Variables

IIFEs help avoid polluting the global scope by keeping variables local to the function:

(function() {
    var localVar = "I am local!";
    document.write(localVar);
})();
// Output: I am local!

console.log(localVar);
// Error: localVar is not defined
  • How It Works:
    • The variable localVar is defined inside the IIFE.
    • It cannot be accessed outside the IIFE, keeping the global scope clean.

4. IIFE with Return Value

You can return a value from an IIFE and store it in a variable:

var result = (function(a, b) {
    return a + b;
})(5, 10);

document.write(result);
// Output: 15
  • How It Works:
    • The function adds two numbers and returns the result.
    • It is immediately called with the values 5 and 10.
    • The result 15 is stored in the variable result and written to the document.

5. IIFE for Initialization

IIFEs are often used for initializing variables and creating private variables:

var app = (function() {
    var privateVar = "I am private!";
    var publicVar = "I am public!";

    return {
        getPublicVar: function() {
            return publicVar;
        }
    };
})();

document.write(app.getPublicVar());
// Output: I am public!

console.log(privateVar);
// Error: privateVar is not defined
  • How It Works:
    • The IIFE defines a private variable privateVar and a public variable publicVar.
    • It returns an object with a method getPublicVar that returns publicVar.
    • The private variable privateVar cannot be accessed outside the IIFE.

Why Use IIFEs?

  1. Avoid Global Scope Pollution:
    • By wrapping your code in an IIFE, you prevent variables and functions from being added to the global scope, which can lead to naming conflicts.
  1. Create Private Variables:
    • Variables declared inside an IIFE are inaccessible from outside, allowing you to create private variables and functions.
  1. Execute Code Immediately:
    • IIFEs are useful for running initialization code that only needs to run once, such as setting up configurations or initializing libraries.
  1. Modular Code:
    • IIFEs can be used to create modular code structures, where each module has its own private scope.

Summary

  • IIFE: A function that runs immediately after it is defined.
  • Syntax: (function() { ... })();
  • Avoids Global Scope Pollution: Keeps variables inside the IIFE, preventing conflicts.
  • Parameters: Can take parameters and be immediately called with values.
  • Return Value: Can return values that can be used outside the IIFE.
  • Initialization: Useful for initializing variables and creating private variables.

When to Use IIFEs

  • Initialization: When you need to initialize variables or set up configurations.
  • Avoiding Conflicts: When you want to avoid polluting the global scope with variables.
  • Private Scope: When you need to create private variables or functions that should not be accessible from outside.

IIFEs are a powerful tool in JavaScript for managing scope, avoiding global variable pollution, and executing code immediately. They are particularly useful in larger applications where maintaining clean and modular code is essential. By understanding how to use IIFEs effectively, you can write more organized and maintainable JavaScript code.

Pass by Value

In JavaScript, when you pass a primitive value (like a number, string, or boolean) to a function, the function receives a copy of that value, not the original value itself. This means that any changes made to the parameter inside the function do not affect the original value outside the function.

Key Points

  1. Primitive Types: Numbers, strings, booleans, null, and undefined are passed by value.
  1. Copy of Value: The function gets a copy of the value, not the original value.
  1. No Side Effects: Changes to the parameter inside the function do not affect the original value.

Examples

1. Passing a Number

    num = 20; // This change does not affect the original value
    console.log("Inside function:", num); // Output: 20
}

changeValue(originalNum);
console.log("Outside function:", originalNum); // Output: 10
  • The function changeValue receives a copy of originalNum.
  • Changing num inside the function does not affect originalNum outside the function.

2. Passing a String

    str = "New String"; // This change does not affect the original value
    console.log("Inside function:", str); // Output: New String
}

let originalStr = "Original String";
changeString(originalStr);
console.log("Outside function:", originalStr); // Output: Original String
  • The function changeString receives a copy of originalStr.
  • Changing str inside the function does not affect originalStr outside the function.

3. Passing a Boolean

function changeBoolean(bool) {
    bool = false; // This change does not affect the original value
    console.log("Inside function:", bool); // Output: false
}

let originalBool = true;
changeBoolean(originalBool);
console.log("Outside function:", originalBool); // Output: true
  • The function changeBoolean receives a copy of originalBool.
  • Changing bool inside the function does not affect originalBool outside the function.

Summary

  • Pass by Value: Primitive values (numbers, strings, booleans, null, undefined) are passed by value.
  • Copy of Value: The function receives a copy of the value, not the original value.
  • No Side Effects: Changes to the parameter inside the function do not affect the original value outside the function.

    • Block-scoped: Like let, const is also block-scoped.

    • Hoisted but not initialized: Like let, const variables are hoisted but not initialized. Accessing them before the declaration results in a ReferenceError.

Private Properties and Methods

In JavaScript, private properties and methods are used to encapsulate data within an object or class, making them inaccessible from outside the object or class. This ensures that certain properties or methods cannot be modified or accessed directly, promoting better code organization and security.

The use of var, let, or const inside a constructor function or class can help create private members. Below is an explanation of how private properties and methods work:

1. Using var, let, or const for Private Members

When you declare variables using var, let, or const inside a constructor function or class, they become private because they are scoped to the function or block in which they are defined. These variables are not accessible outside the function or class.

Example Using var:

function Mobile() {
    var price = 3000; // Private property

    this.getPrice = function() {
        return price; // Accessible via a public method
    };

    this.setPrice = function(newPrice) {
        price = newPrice; // Modify private property via a public method
    };
}

var samsung = new Mobile();
console.log(samsung.price); // undefined (price is private)
console.log(samsung.getPrice()); // 3000 (access via public method)
samsung.setPrice(4000);
console.log(samsung.getPrice()); // 4000 (updated value)
  • Here, price is a private property because it is declared with var inside the constructor.
  • The getPrice and setPrice methods act as public interfaces to access and modify the private property.

Example Using let or const:

function Mobile() {
    let price = 3000; // Private property using let
    const tax = 500;  // Private constant using const

    this.getTotalPrice = function() {
        return price + tax; // Access private members
    };

    this.setPrice = function(newPrice) {
        price = newPrice; // Modify private property
    };
}

var samsung = new Mobile();
console.log(samsung.price); // undefined (price is private)
console.log(samsung.tax);   // undefined (tax is private)
console.log(samsung.getTotalPrice()); // 3500 (access via public method)
samsung.setPrice(4000);
console.log(samsung.getTotalPrice()); // 4500 (updated value)
  • let and const behave similarly to var but follow block scoping rules.
  • const is used for values that should not change (constants).

2. Private Methods

Just like private properties, private methods can also be created using var, let, or const. These methods are not accessible outside the constructor or class.

Example:

function Mobile() {
    var price = 3000; // Private property

    var calculateDiscount = function(discount) {
        return price - discount; // Private method
    };

    this.getDiscountedPrice = function(discount) {
        return calculateDiscount(discount); // Call private method
    };
}

var samsung = new Mobile();
console.log(samsung.calculateDiscount(500)); // Error: calculateDiscount is private
console.log(samsung.getDiscountedPrice(500)); // 2500 (access via public method)
  • The calculateDiscount method is private because it is declared with var.
  • It can only be accessed indirectly through the public method getDiscountedPrice.

concept of a prototype

In JavaScript, the concept of a prototype is fundamental to understanding how objects inherit properties and methods. Every object in JavaScript has an internal prototype, which is a reference to another object that provides a template for the properties and methods that the object will have.

Understanding Prototypes

  1. Prototype Chain: Each object has a prototype, and that prototype is itself an object, which may have its own prototype, and so on. This forms a chain of prototypes, known as the prototype chain.
  1. Inheritance: Prototypes enable inheritance in JavaScript. When you try to access a property or method on an object, JavaScript will first look for it on the object itself. If it doesn't find it, it will look for it on the object's prototype, and then on the prototype's prototype, and so on, until it reaches the end of the prototype chain.

Example of Prototypes

Let's look at an example to illustrate how prototypes work:

// Constructor function
function Mobile(model_no, sprice) {
    this.model = model_no;
    this.color = "white";
    this.price = 3000;
    this.sp = sprice;
}

// Adding a method to the prototype
Mobile.prototype.sellingprice = function() {
    return this.price + this.sp;
};

// Creating instances
var samsung = new Mobile("Galaxy", 2000);
var nokia = new Mobile("3310", 1000);

// Accessing the method via the prototype
console.log(samsung.sellingprice()); // Output: 5000
console.log(nokia.sellingprice()); // Output: 4000

Explanation

  1. Constructor Function: The Mobile function is a constructor that initializes new objects with properties model, color, price, and sp.
  1. Prototype Method: The sellingprice method is added to the Mobile prototype. This means that all instances of Mobile will have access to this method.
  1. Creating Instances: Two instances, samsung and nokia, are created using the Mobile constructor.
  1. Accessing the Method: The sellingprice method is accessed via the prototype. When samsung.sellingprice() or nokia.sellingprice() is called, JavaScript looks for the sellingprice method on the object's prototype.

Benefits of Prototypes

  • Memory Efficiency: Methods defined on the prototype are shared among all instances, reducing memory usage.
  • Inheritance: Prototypes allow objects to inherit properties and methods from other objects, promoting code reuse.

A class expression is one of the two ways to define a class in JavaScript (the other being a class declaration). A class expression allows you to define a class and assign it to a variable. It can be either named or unnamed.

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a class to inherit properties and methods from another class. This promotes code reuse, reduces redundancy, and establishes a hierarchical relationship between classes. In JavaScript, inheritance is achieved using the extends keyword with ES6 classes.

Key Concepts of Inheritance

  1. Child Class (Subclass): The class that inherits from the parent class.

    • An object galaxy is created from the Samsung class.
    • The displayBrand method is inherited from the Mobile class.

Method overriding

is a concept in object-oriented programming (OOP) where a subclass (child class) provides a specific implementation for a method that is already defined in its superclass (parent class). This allows the subclass to override the behavior of the method inherited from the superclass.

Key Concepts of Method Overriding

  1. Superclass (Parent Class): The class that defines the original method.
  1. Subclass (Child Class): The class that inherits from the superclass and provides a new implementation of the method.
  1. Method Overriding: The process of providing a new implementation of a method in the subclass that has the same name, return type, and parameters as the method in the superclass.

Example of Method Overriding in JavaScript

Let's illustrate method overriding with an example:

// Superclass (Parent Class)
class Mobile {
    constructor(brand) {
        this.brand = brand;
    }

        console.log("Brand: " + this.brand);
    }
}

// Subclass (Child Class) inheriting from Mobile
class Samsung extends Mobile {
    constructor(brand, model) {
        super(brand); // Call the parent class constructor
        this.model = model;
    }

    // Overriding the displayInfo method
    displayInfo() {
        super.displayInfo(); // Call the parent class method
        console.log("Model: " + this.model);
    }
}

// Creating an object from the Samsung class
const galaxy = new Samsung("Samsung", "Galaxy S21");
galaxy.displayInfo();
// Output:
// Brand: Samsung
// Model: Galaxy S21

Explanation

  1. Superclass (Mobile):
    • Defines a constructor that initializes the brand property.
    • Has a method displayInfo that displays the brand.
  1. Subclass (Samsung):
    • Inherits from the Mobile class using the extends keyword.
    • Defines its own constructor that calls the parent class constructor using super(brand) and initializes an additional property model.
    • Overrides the displayInfo method to provide a new implementation.

  1. Creating an Object:
    • An object galaxy is created from the Samsung class.
    • When the displayInfo method is called on the galaxy object, it uses the overridden method in the Samsung class, which first calls the parent class's displayInfo method and then adds its own functionality.

Benefits of Method Overriding

  1. Customization: Allows subclasses to provide specific implementations of methods tailored to their needs.
  1. Polymorphism: Enables polymorphic behavior, where a method call can exhibit different behaviors depending on the object it is acting upon.
  1. Code Reuse: Promotes code reuse by allowing subclasses to inherit and customize behavior from superclasses.

Rules for Method Overriding

  1. Same Method Name: The method in the subclass must have the same name as the method in the superclass.
  1. Same Parameters: The method in the subclass must have the same parameters as the method in the superclass.
  1. Same Return Type: The method in the subclass must have the same return type as the method in the superclass (though JavaScript is more flexible with return types).

The join() method takes an optional argument called the separator, which determines how the array elements are joined together in the resulting string.

Syntax:

The reverse() method in JavaScript is used to reverse the order of elements in an array. It modifies the original array in place, meaning that the array is reversed directly, and no new array is created.

reverse() Method

The reverse() method reverses the order of the elements in an array. The first element becomes the last, the second element becomes the second-to-last, and so on.

Syntax:

array.reverse();
  • array: The array whose elements you want to reverse.
  • No return value needed: The method modifies the original array and also returns the reversed array.

fruits.reverse();
